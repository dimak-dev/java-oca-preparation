== Назначение

Тема "Java Basics" является фундаментальной для экзамена OCA и покрывает основные концепции языка Java.
Понимание структуры программы, правил написания классов, метода main и области видимости критически важно для успешной сдачи экзамена.
Эта тема составляет основу для всех остальных разделов экзамена.

== Теория

=== Структура Java программы

Java программа состоит из одного или нескольких классов, заключенных в пакеты (packages).
Каждый исходный файл Java должен иметь расширение `.java` и может содержать только один публичный класс, имя которого должно совпадать с именем файла.

Основные элементы структуры:

* **Package declaration** - объявление пакета (опционально)
* **Import statements** - импорт классов и пакетов (опционально)
* **Class declaration** - объявление класса
* **Class body** - тело класса с полями и методами

=== Правила написания классов

Правила именования и объявления классов в Java:

* Имя класса должно начинаться с заглавной буквы
* Используется PascalCase для составных имен (например, `MyClassName`)
* Имя публичного класса должно совпадать с именем файла
* В одном файле может быть только один публичный класс
* Классы без модификатора доступа имеют package-private видимость

=== Метод main

Метод `main` - точка входа в Java приложение:

[source,java]
----
public static void main(String[] args)
----

Характеристики метода main:

* Должен быть `public` - доступен извне
* Должен быть `static` - вызывается без создания экземпляра
* Возвращает `void` - ничего не возвращает
* Принимает массив `String[]` в качестве параметра
* Имя параметра может быть любым, но `args` - общепринятая практика

=== Область видимости

Область видимости определяет, где переменные и методы могут быть использованы:

**Модификаторы доступа:**

* `public` - доступен отовсюду
* `protected` - доступен в пакете и наследниках
* package-private (без модификатора) - доступен только в пакете
* `private` - доступен только в классе

**Области видимости переменных:**

* **Локальные переменные** - объявлены в методах, существуют только в рамках метода
* **Поля класса** - объявлены в классе, доступны всем методам класса
* **Параметры методов** - существуют только в рамках метода

=== Ключевые синтаксические элементы

Основные синтаксические элементы Java:

* **Фигурные скобки `{}`** - определяют блоки кода
* **Круглые скобки `()`** - для параметров методов и выражений
* **Точка с запятой `;`** - завершает операторы
* **Ключевые слова** - зарезервированные слова Java (`class`, `public`, `static`, etc.)
* **Идентификаторы** - имена переменных, методов, классов
* **Литералы** - константные значения в коде
* **Комментарии** - `//` для однострочных, `/* */` для многострочных

== Примеры вопросов

=== Вопрос 1
Какая из следующих сигнатур метода `main` является правильной для запуска Java приложения?

* A) `public static void main(String[] args)`
* B) `public void main(String[] args)` ❌ НЕВЕРНО
* C) `static void main(String[] args)` ❌ НЕВЕРНО
* D) `public static int main(String[] args)` ❌ НЕВЕРНО

*Правильный ответ:* A

*Объяснение:* Метод `main` должен быть `public` (доступен извне), `static` (вызывается без создания экземпляра), возвращать `void` и принимать массив `String[]`.

=== Вопрос 2
Что произойдет при попытке компиляции следующего кода?

[source,java]
----
public class Test {
    public static void main(String[] args) {
        String name;
        System.out.println(name);
    }
}
----

* A) Код скомпилируется и выведет `null` ❌ НЕВЕРНО
* B) Ошибка компиляции: локальная переменная не инициализирована
* C) Код скомпилируется и выведет пустую строку ❌ НЕВЕРНО
* D) Ошибка выполнения: NullPointerException ❌ НЕВЕРНО

*Правильный ответ:* B

*Объяснение:* Локальные переменные должны быть инициализированы перед использованием. Компилятор выдаст ошибку "variable name might not have been initialized".

=== Вопрос 3
В каком файле должен находиться следующий класс?

[source,java]
----
public class MyApplication {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
----

* A) Любой файл с расширением .java ❌ НЕВЕРНО
* B) application.java ❌ НЕВЕРНО
* C) MyApplication.java
* D) myapplication.java ❌ НЕВЕРНО

*Правильный ответ:* C

*Объяснение:* Имя файла должно точно совпадать с именем публичного класса, включая регистр символов.

== Типичные ошибки

=== Ошибка 1: Неправильная сигнатура метода main

**Проблема:** Использование неправильной сигнатуры метода `main`, например:
* `public void main(String[] args)` - отсутствует `static`
* `public static void main(String args)` - параметр не массив
* `public static void Main(String[] args)` - неправильный регистр

**Решение:** Всегда используйте точную сигнатуру: `public static void main(String[] args)`

=== Ошибка 2: Использование неинициализированных локальных переменных

**Проблема:** Попытка использовать локальную переменную до её инициализации:
[source,java]
----
int count;
System.out.println(count); // ОШИБКА КОМПИЛЯЦИИ!
----

**Решение:** Всегда инициализируйте локальные переменные перед использованием:
[source,java]
----
int count = 0; // или любое другое подходящее значение
System.out.println(count); // OK
----

=== Ошибка 3: Несоответствие имени класса и файла

**Проблема:** Имя публичного класса не совпадает с именем файла:
* Файл: `myclass.java`, класс: `public class MyClass` - разный регистр
* Файл: `Test.java`, класс: `public class TestClass` - разные имена

**Решение:** Имя файла должно точно совпадать с именем публичного класса, включая регистр символов.

=== Ошибка 4: Путаница между статическими и нестатическими членами

**Проблема:** Попытка обращения к нестатическим членам из статического контекста:
[source,java]
----
public class Test {
    String message = "Hello";

    public static void main(String[] args) {
        System.out.println(message); // ОШИБКА!
    }
}
----

**Решение:** Создайте экземпляр класса или сделайте член статическим.

* **Reflection API** - получение информации о классах во время выполнения
* **java.lang.Class** - метаинформация о классах
